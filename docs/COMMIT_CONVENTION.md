# Соглашение о коммитах

Используем [Conventional Commits](https://www.conventionalcommits.org/) на русском языке.

## Формат

```
<тип>(<область>): <описание>

[тело]

[футер]
```

## Типы коммитов

| Тип | Описание |
|-----|----------|
| `feat` | Новая функциональность |
| `fix` | Исправление бага |
| `docs` | Изменения в документации |
| `style` | Форматирование, пробелы, точки с запятой |
| `refactor` | Рефакторинг без изменения функциональности |
| `perf` | Улучшение производительности |
| `test` | Добавление или исправление тестов |
| `build` | Сборка, зависимости, CI/CD |
| `chore` | Рутинные задачи, настройка инструментов |

## Примеры

```
feat(auth): добавить авторизацию через Google

fix: исправить отображение модального окна на мобильных

docs: обновить README с инструкцией по установке

refactor(api): упростить обработку ошибок

style: привести код к единому стилю форматирования

chore: обновить зависимости
```

## Рекомендации

- Описание начинается с маленькой буквы
- Без точки в конце описания
- Используй повелительное наклонение: "добавить", "исправить", "обновить"
- Область (`scope`) опциональна
- Тело коммита для подробностей (опционально)
